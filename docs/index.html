🏗️ Arquitectura de Despliegue y SPA

El sistema implementa una arquitectura desacoplada donde el Frontend es una Single Page Application (SPA) y el Backend es una API REST Stateless.

[Cliente Browser]
↓ (HTTPS / JSON)
[Netlify: Frontend Vue 3] ➔ [_redirects config] ➔ [Routing Interno]
↓ (API Calls)
[Railway: Backend Node.js] ➔ [CORS Headers] ➔ [Mongoose ODM]
↓ (TCP/TLS)
[MongoDB Atlas: Database]

Configuración de SPA y Headers

  • Redirecciones: Se implementó un archivo _redirects en Netlify para redirigir todas las peticiones a index.html, evitando errores 404 al recargar rutas internas de Vue Router.
  • CORS: El servidor Express utiliza el middleware cors configurado específicamente para el dominio de producción de Netlify, evitando accesos no autorizados.
  • Headers: Implementación de headers de seguridad básicos y control de caché para optimizar la carga de activos estáticos.

🔍 Diagnóstico y Monitoreo

Uso de Logs para Diagnóstico

Para garantizar la estabilidad en producción, se implementó un sistema de logging en el backend:

  • Logs de Acceso: Registro de cada petición HTTP (Método, Ruta, Tiempo de respuesta, Status Code) mediante middleware.
  • Logs de Error: Captura de excepciones en el middleware de errores global, registrando el stack trace en la consola de Railway para depuración rápida.
  • Monitoreo de DB: Logs de conexión y desconexión de MongoDB Atlas para detectar latencias o caídas del clúster.

⚡ Rendimiento y Optimización

Se realizaron pruebas de rendimiento utilizando Google Lighthouse en el entorno de producción de Netlify.

Mejoras Implementadas:

  • Minificación: Uso de Vite para el bundling y minificación de JS/CSS.
  • Lazy Loading: Carga diferida de componentes de Vue para reducir el tamaño del bundle inicial.
  • Caché: Implementación de estrategias de caché en el navegador para activos estáticos.
