MercApp - Micrositio Técnico

Sistema de Gestión de Inventario para la Unidad 4 - Aplicaciones Web

🏗️ Arquitectura de Despliegue

El sistema sigue una arquitectura de desacoplamiento total entre el cliente y el servidor (Decoupled Architecture), permitiendo escalabilidad independiente.

Frontend (Netlify) ➔ API Backend (Railway) ➔ Base de Datos (MongoDB Atlas)
  • Frontend: SPA desarrollada en Vue 3, desplegada en Netlify mediante el build de Vite.
  • Backend: API REST desarrollada en Node.js/Express, desplegada en Railway utilizando Nixpacks.
  • Base de Datos: MongoDB Atlas (Cluster M0), proporcionando persistencia de datos en la nube con TLS.
  • Seguridad: Implementación de CORS para restringir el acceso únicamente al dominio de Netlify.

🚀 Guía de Ejecución Local

Para ejecutar el proyecto localmente, siga estos pasos:

1. Backend

  1. cd backend (o carpeta raíz según estructura)
  2. npm install
  3. Configurar .env con MONGODB_URI y PORT=3000
  4. npm run dev

2. Frontend

  1. cd frontend
  2. npm install
  3. Configurar .env con VITE_API_URL=http://localhost:3000/api
  4. npm run dev

📡 Documentación de Endpoints

Método Ruta Descripción Ejemplo Response
GET /api/health Verifica el estado del servidor y BD { "status": "up", "db": "connected" }
GET /api/products Lista todos los productos [{ "id": "...", "name": "Producto A" }]

PUT /api/products/:id Actualiza un producto existente { "message": "Actualizado" }
DELETE /api/products/:id Elimina un producto { "message": "Eliminado" }
